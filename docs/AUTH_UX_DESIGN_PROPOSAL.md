# 登录注册交互设计方案

## 需求分析

### 功能矩阵

| 地区 | 登录方式 | 认证方式 |
|------|---------|---------|
| 国内 | 邮箱 | 验证码 / 密码 |
| 国内 | 手机号 | 验证码 / 密码 |
| 国外 | 邮箱 | 验证码 / 密码 |
| 国外 | OAuth | GitHub / Google / Microsoft |

### 设计目标

1. **简洁直观** - 用户能快速理解如何登录
2. **减少选择** - 避免过多选项造成决策疲劳
3. **智能适配** - 根据地区自动显示合适的登录方式
4. **灵活切换** - 支持在不同认证方式间切换
5. **一致体验** - 注册和登录流程保持一致

## 边界情况处理

### 1. 账号不存在
```
用户输入邮箱 → 点击继续
↓
系统检测：账号不存在
↓
提示："该邮箱尚未注册，是否立即注册？"
[返回] [立即注册]
```

### 2. 验证码发送失败
```
显示错误提示："验证码发送失败，请稍后重试"
[重新发送] [使用密码登录]
```

### 3. 密码错误
```
显示错误提示："密码错误，请重试"
[忘记密码？] → 跳转到重置密码流程
```

### 4. 验证码错误
```
显示错误提示："验证码错误或已过期"
[重新发送验证码]
```

### 5. OAuth 登录失败
```
显示错误提示："第三方登录失败，请重试或使用其他方式"
[返回] [使用邮箱登录]
```

## 移动端适配

### 响应式设计要点

1. **大按钮**：移动端按钮高度至少 44px
2. **大输入框**：避免缩放，font-size 至少 16px
3. **自动聚焦**：进入页面自动聚焦输入框
4. **键盘优化**：
   - 邮箱输入框：`type="email"`
   - 手机输入框：`type="tel"`
   - 验证码输入框：`type="number" inputmode="numeric"`

## 安全性考虑

### 1. 验证码安全
- 验证码有效期：5 分钟
- 同一账号 1 分钟内最多发送 1 次
- 同一 IP 1 小时内最多发送 10 次
- 验证码错误 5 次后锁定 15 分钟

### 2. 密码安全
- 密码错误 5 次后锁定账号
- 显示密码强度提示
- 支持密码可见性切换

### 3. OAuth 安全
- 使用 PKCE 流程
- 验证 state 参数
- 限制回调域名

## 总结

### 推荐方案：智能单页式 + 分步引导

**理由**：
1. ✅ 首屏简洁，降低用户决策成本
2. ✅ 智能识别，减少用户操作
3. ✅ 分步引导，每步聚焦一个任务
4. ✅ 现代化体验，符合主流产品趋势
5. ✅ 易于扩展，支持未来新增登录方式

**关键特性**：
- 一个输入框搞定邮箱/手机号
- 自动检测地区，显示对应登录方式
- 验证码优先，密码作为备选
- OAuth 作为快捷选项（国外）
- 清晰的错误提示和引导

**用户体验**：
- 新用户：3 步完成注册（输入 → 验证 → 完成）
- 老用户：2 步完成登录（输入 → 验证）
- 最快路径：OAuth 一键登录（国外）
